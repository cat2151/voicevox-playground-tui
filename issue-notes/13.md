# issue リグレッションにより、「現在行にcacheが獲得できたあと、裏で、cacheのない行を探索してcacheを取得する機能」が動作しなくなってしまった #13
[issues #13](https://github.com/cat2151/voicevox-playground-tui/issues/13)

# なにが困るの？
- これまでは、放置していれば、見えている範囲のすべての行のcacheが自動ですべて取得され、カーソル移動だけで即座にplayできた
- 現在は、リグレッションにより、見えている範囲の行のcacheの獲得を手動でしなければならなくなっており、即座のplayが実現できず、UXが低下した

# 応急対策は？
- 上記機能を改めて実装する
- testを書く
- testにより、リグレッションが検知できるし、contextが改善される、と考える

# 根本対策は？
- リグレッションにより機能喪失したことは、以下の問題を示唆するアラートである、と判断する
  - coding agentのアクセスするcontextに、「現在行にcacheが獲得できたあと、裏で、cacheのない行を探索してcacheを取得する機能」がない
- 対策
  - 専用ソースに切り出す。これにより、contextが改善される、と考える
  - もしそれでもダメな場合の対策案としては、copilot_instructions.mdに、上記の機能を明示する
