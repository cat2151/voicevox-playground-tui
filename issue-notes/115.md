# issue config.toml にVOICEVOXの場所を設定可能にする #115
[issues #115](https://github.com/cat2151/voicevox-playground-tui/issues/115)

- これまでの課題
    - VOICEVOX-engineをデフォルトと違う場所にinstallしていると、
    - アプリ起動時のVOICEVOX-engine起動に失敗し、
    - userが混乱するメッセージが表示される
    - メッセージ：
        - `VOICEVOXをインストールしてから再度お試しください。`
        - なぜuserが混乱するか？
            - userはVOICEVOXを、もうinstallしてある。
            - install場所を、アプリが知らないだけである
- 対策
    - 現在historyを保存しているdirに、config.tomlを保存する
    - 存在しない場合はtomlファイルを生成する
        - 生成内容は、コメントで、
        - `voicevox_path = "<your voicevox path>`
        - `voicevox_nemo_path = "<your voicevox nemo path>"`
    - Windowsなら、VOICEVOX-engineのフルパスは、
        - voicevox_path + "/vv-engine/run.exe" となる
        - これがserverとして実行するfilenameである
